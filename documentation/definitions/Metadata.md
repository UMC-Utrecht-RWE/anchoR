> The study-variable specification: one row per `variable_id x window_name`, describing which concept to look for, where the window sits relative to the anchor, and how to collapse matches.

Metadata is the "recipe" table anchoR cross-joins against [Population](Population.md). Each row says: for this `variable_id`, look for `concept_id` inside a window built by a [Constructor](Constructor.md) (named in the `constructor` column), and collapse whatever [concept](Concepts.md) records land in that window using a [Selector](Selector.md).

Mandatory columns after normalisation: `variable_id`, `concept_id`, `window_name`, `constructor`, `selector`, `start_offset`/`end_offset`, `anchor_start_col`/`anchor_end_col` (aliased from `anchor_date_start`/`anchor_date_end`), `range_min`/`range_max`, `event_col`, `end_cap_offset`, `start_look_back`/`end_look_back` (see [IN_PRIOR_PREG](IN_PRIOR_PREG.md), unrelated to `start_offset`/`end_offset` despite the similar name). `normalize_metadata()` fills in sensible defaults for anything not supplied, so a minimal metadata table only needs `variable_id`, `concept_id`, `constructor`, `selector`, and `start_offset`/`end_offset`.
