> [Engine](<Episode-Based Window Engine.md>) configured with `event_select = "PRIOR"`, `end_boundary = "event_END"`: one window per episode that ended before the anchor date.

Selects *every* episode with `event_end < anchor`,  not just the most recent one, so a person with several prior pregnancies gets one candidate [Window](Window.md) per prior pregnancy, and the [Selector](Selector.md) then picks across all of them (e.g. `LATEST` returns the latest matching record from *any* of those windows).

Window: `[episode_start + start_offset, episode_end + end_offset]`, optionally capped by [`end_cap_offset`](<End Cap Offset.md>) (`window_end = pmin(episode_end + end_offset, episode_start + end_cap_offset)`). The capped variant is how a study variable like "records during the first N days of any prior pregnancy" is expressed, same constructor, different offset metadata, not new code.

Optionally, metadata can also set `start_look_back`/`end_look_back` (separate columns from `start_offset`/`end_offset`, both NA by default) to restrict *which prior pregnancies are eligible at all*: only episodes overlapping the anchor-relative range `[anchor + start_look_back, anchor + end_look_back]` are considered, e.g. "only prior pregnancies within the last 10 years" is `start_look_back = -3652, end_look_back = 0`. This is a selection filter on the episode itself (same overlap rule as [OUTSIDE_ALL_PREG](OUTSIDE_ALL_PREG.md)'s search range), not a clip on the resulting window: a prior pregnancy entirely outside the lookback range is dropped from consideration before any window is built, while one that overlaps is kept and its window is still computed from `start_offset`/`end_offset` exactly as described above, unclipped, even if part of that window falls outside the lookback range. These are kept separate from `start_offset`/`end_offset` on purpose: those already shift the episode's own start/end (and can be positive, e.g. "start 90 days into the pregnancy"), and reusing them for an anchor-relative bound would mean a positive `start_offset` pushes the lookback's lower bound past the anchor, which a by-definition-prior episode could never satisfy. Note: despite the similar name, `start_look_back`/`end_look_back` are unrelated to `start_offset`/`end_offset` (which used to accept `start_look_back`/`end_look_back` as alternate names for themselves in older versions of the package -- that alias no longer exists).
