> Cross-joins [[Population]] with [[Metadata]] and computes one [[Window]] per population row × metadata row, applying each row's [[Window Constructor]]; the step [[anchor()]] runs before any [[Selector|selector]] SQL executes.

Steps: `validate_anchor_inputs()` normalizes inputs → `cross_join_population_metadata()` builds the cartesian product (with a fast path avoiding the cartesian merge overhead when metadata has one row and no overlapping column names) → each row gets a stable `.window_row_id` to preserve pre-join order → `apply_window_constructors()` runs one constructor per unique `constructor` value and row-binds the results (a constructor may expand one row into several, as the episode-based ones do) → `finalize_windows()` restores original row order, marks `window_valid` (`TRUE` only when both bounds are non-missing and `window_start <= window_end`), and assigns a synthetic `anchor_row_id` used as a stable key by the downstream SQL layer.
